TOPOLOGI JARINGAN

Tampilan topologi di gns3

Topologi jaringan yang digunakan untuk implementasi DNS di MikroTik terdiri dari komponen-komponen berikut:

Router MikroTik berfungsi sebagai DNS server dan DNS cache untuk seluruh jaringan lokal. Semua klien dalam jaringan akan menggunakan router sebagai DNS resolver utama.

Berikut adalah komponen-komponen dalam topologi jaringan DNS:

1. MikroTik Router (192.168.88.1)
# Berfungsi sebagai DNS server dan DNS cache
2. Internet/ISP → Sumber DNS Upstream
# Menyediakan resolusi DNS untuk domain publik (8.8.8.8, 1.1.1.1)
3. Jaringan LAN (192.168.88.0/24)
# Perangkat klien yang menggunakan layanan DNS MikroTik
4. Server Internal (192.168.88.254)
# Server internal yang di-resolve melalui Static DNS

Dalam topologi ini, MikroTik Router menangani semua permintaan DNS dari jaringan lokal. Untuk domain publik (seperti google.com), router akan meneruskan permintaan ke server DNS upstream (seperti 8.8.8.8). Untuk domain internal (seperti server.local), router akan menggunakan entri static DNS yang telah dikonfigurasi.

Router MikroTik harus dikonfigurasi dengan benar untuk membatasi akses DNS hanya untuk jaringan internal. Ini mencegah router digunakan sebagai open DNS resolver yang dapat dimanfaatkan untuk serangan DNS amplification.

Dengan menggunakan topologi ini, kita dapat mengoptimalkan kinerja DNS dengan memanfaatkan fitur static DNS dan DNS cache yang ditawarkan oleh RouterOS.

1. PENGERTIAN INTERFACE LIST DI MIKROTIK

PENGERTIAN INTERFACE LIST DI MIKROTIK

Dari sumber MikroTik Documentation (2023), Interface List adalah fitur yang memungkinkan pendefinisian sekumpulan antarmuka untuk memudahkan pengelolaan antarmuka pada berbagai konfigurasi berbasis antarmuka seperti Neighbor Discovery, Firewall, Bridge, dan Internet Detect.

Untuk melihat daftar interface list yang ada di router, gunakan perintah berikut:

/interface list print
# Menampilkan semua interface list yang tersedia

Dari sumber MikroTik Wiki (2022), pada router MikroTik terdapat beberapa daftar yang sudah ditentukan sebelumnya - all (berisi semua antarmuka), none (tidak berisi antarmuka), dynamic (berisi antarmuka dinamis), dan static (berisi antarmuka statis). Selain itu, dimungkinkan untuk membuat daftar antarmuka tambahan sesuai kebutuhan.

Interface list sangat bermanfaat saat Anda perlu mengelompokkan beberapa interface untuk diterapkan konfigurasi yang sama, seperti aturan firewall yang sama untuk semua interface dalam satu kelompok.

Untuk membuat interface list baru melalui CLI, gunakan perintah:

/interface list add name=nama_list
# Membuat interface list baru

Dari sumber Manito Networks (2018), Interface List di MikroTik dapat digunakan untuk mengelompokkan antarmuka berdasarkan zona kepercayaan jaringan. Contohnya membuat zone "Trusted" untuk jaringan internal yang aman, "Semi-Trusted" untuk jaringan dengan tingkat kepercayaan menengah, dan "Untrusted" untuk koneksi ke internet.

Dari 3 pendapat tersebut dapat disimpulkan bahwa Interface List di MikroTik adalah fitur yang memungkinkan pengelompokan beberapa antarmuka ke dalam satu daftar untuk memudahkan pengelolaan dan konfigurasi, terutama pada bagian yang berhubungan dengan Firewall, Bridge, dan fitur networking lainnya.

2. MENGELOLA ANGGOTA INTERFACE LIST

ENGELOLA ANGGOTA INTERFACE LIST

Dari sumber MikroTik Documentation (2023), sub-menu interface list member berisi informasi tentang anggota antarmuka yang dikonfigurasi secara statis untuk setiap daftar antarmuka. Perlu dicatat bahwa antarmuka yang ditambahkan secara dinamis oleh pernyataan include dan exclude tidak direpresentasikan dalam sub-menu ini.

Untuk menambahkan interface ke dalam interface list melalui CLI, gunakan perintah berikut:

/interface list member add list=nama_list interface=nama_interface
# Menambahkan interface ke dalam list

Contoh menambahkan beberapa interface sekaligus ke list LAN:

/interface list member add list=LAN interface=ether2
/interface list member add list=LAN interface=ether3
/interface list member add list=LAN interface=ether4
# Menambahkan ether2, ether3, dan ether4 ke list LAN

Dari sumber MikroTik Forum (2018), satu interface dapat menjadi anggota dari beberapa list berbeda yang dapat digunakan untuk tujuan yang berbeda. Misalnya, satu interface dapat menjadi bagian dari dua list, satu untuk konfigurasi firewall dan yang lainnya untuk konfigurasi MAC server.

Untuk melihat semua anggota interface list, gunakan perintah:

/interface list member print
# Menampilkan semua anggota interface list

Untuk menghapus interface dari interface list, Anda perlu mengetahui nomor anggota dalam list, yang bisa didapatkan dari perintah print, kemudian gunakan perintah:

/interface list member remove numbers=0
# Menghapus interface dengan nomor 0 dari list

Pastikan untuk memeriksa kembali semua konfigurasi yang bergantung pada interface list setelah melakukan perubahan, terutama pada aturan firewall dan bridge configuration.

Dari 3 pendapat tersebut dapat disimpulkan bahwa pengelolaan anggota Interface List di MikroTik dilakukan melalui sub-menu interface list member, dengan perintah-perintah untuk menambahkan, melihat, dan menghapus interface dari list. Setiap interface dapat menjadi anggota dari beberapa list berbeda sesuai kebutuhan konfigurasi.
